# GapHarness Master Plan — From "Reject" to Publishable

Reconciled across 6 solution designs and 3 adversarial critiques. Where critics caught the designs being circular or over-optimistic, the plan below adopts the critic's fix, not the design's original framing. The non-negotiable spine: **the contribution is a framing (obligation-typed, independently-checkable refusal certificate), and the whole thing stands or falls on one premise — that abstract obligations are a human-reproducible abstraction. That premise is tested FIRST, cheaply, with a pre-committed stop-loss, before any expensive build.**

---

## PART 1 — THE REFRAMED CORE CLAIM

**One-sentence contribution (survives the "just set cover" attack):**

> GapHarness reframes agent runtime-support selection as a *decidable pre-execution typing problem*: it lifts a request to an abstract obligation profile, then emits a **proof-carrying harness certificate** that a third party can verify in linear time *without trusting the compiler or the LLM* — either witnessing that a declared-affordance set discharges every obligation, or witnessing that none can and naming the missing affordance — turning "did the agent have what it needed?" and "should it have refused?" from post-hoc execution outcomes into statically checkable, registry-relative guarantees.

The optimizer is conceded as textbook weighted set cover + monotone dependency closure (cite Chvátal/Karp); we make zero algorithmic claim. The novelty is the **certificate-as-contract between profiling and execution** — unoccupied in the agent-harness literature (AgentSpec/AutoHarness *enforce at runtime*; NLAH/Harness-Bench *describe/measure*; routers *select tools*).

**The money experiment (reconciled — NOT Design #1's tautological version):**

Design #1's "0 side-effects by construction" is, as all three critics flagged, `f(x)==0` by definition of "pre-execution" — and it needs a live agent stack the repo does not have. The defensible money experiment relocates the win to where it is *not* tautological:

> **The Disguised-Refusal experiment.** On a benchmark of genuinely-unsupported requests whose unsupportedness is *lexically disguised* (scope-confusion phrasings where keyword heuristics fail), measure, under a **non-leaky** verifier feedback channel, and *including a pre-execution-refusal baseline*: (1) refusal precision/recall against **independent** human gold, (2) **side-effects fired before refusal** on a side-effect-logging executor, and (3) **witness checkability** by an independent linear-time verifier. The claim is proven only if GapHarness's *profiler actually catches disguised cases* AND its certificate witness is independently checkable, while feedback baselines either fire side-effects discovering unsupportedness or cannot produce a checkable witness — **and a pre-execution-refusal LLM baseline that lacks the certificate still cannot produce the checkable minimal witness.** If the GapHarness profiler fails on disguised cases as often as baselines, the experiment honestly reports "no win" and we fall back to the auditability/witness claim alone.

This version cannot be dismissed as tautological because the win now depends on a *measured* profiler capability on adversarial inputs, scored against *independent* gold, with the strongest possible baseline (pre-execution refusal) included.
